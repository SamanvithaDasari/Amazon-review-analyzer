API documentation

RESTful endpoints for sentiment analysis, review querying, and aggregate statistics. Backed by FastAPI with an auto-generated OpenAPI spec — also explorable through Swagger UI.

FastAPI JSON Pydantic validation OpenAPI 3.1
GET /api/health Liveness probe

Returns service health. No parameters.

# Response
{ "status": "ok" }
POST /api/sentiment Classify free-text review

Run sentiment analysis on arbitrary text. Returns label, compound score, and probability distribution. Two models supported: lexicon-based VADER (default, fast) and DistilBERT-SST2 (transformer, slower).

# Request
{
  "review_text": "Camera is amazing but battery dies fast.",
  "model": "vader"  // or "hf"
}

# Response
{
  "sentiment": "positive",
  "compound_score": 0.625,
  "positive": 0.412,
  "negative": 0.187,
  "neutral": 0.401,
  "model": "vader"
}

# Try it
curl -X POST http://localhost:8000/api/sentiment \
  -H "Content-Type: application/json" \
  -d '{"review_text": "Phone is great"}'
GET /api/reviews Query the corpus

Paginated review listing with optional filters.

Query parameters:

  • rating — integer 1–5
  • color — string (NULL for this product; see Browse page)
  • storage — string (NULL for this product)
  • limit — integer, default 20
  • offset — integer, default 0
# Example
curl "http://localhost:8000/api/reviews?rating=5&limit=3"

# Response
{
  "total": 163,
  "limit": 3,
  "offset": 0,
  "reviews": [ ... ]
}
GET /api/stats Aggregated insights

The full analytics payload powering the dashboard. Returns sentiment & rating distributions, top TF-IDF keywords, and aspect-based sentiment across 9 product features (camera, battery, display, performance, build, price, heat, delivery, ecosystem).

# Response shape
{
  "total_reviews": 652,
  "average_rating": 3.55,
  "sentiment_distribution": { "positive": 412, "negative": 168, ... },
  "rating_distribution": { "5": 163, "4": 159, ... },
  "top_keywords": [ { "term": "camera", "score": 0.0823 }, ... ],
  "aspects": [
    { "aspect": "camera", "mentions": 187, "positive": 132, ... }
  ]
}

How it's built

Two-layer architecture: FastAPI on an internal port handles sentiment scoring and DB queries with Pydantic-validated request/response models; Flask serves the human-facing UI plus proxy routes to the API. SQLite stores normalized reviews; sentiment is computed at ingestion time. Aspect analysis uses sentence-level VADER scoring across a hand-curated lexicon for 9 product features. Reviews scraped via Playwright with cookie-based authentication after multiple data-acquisition paths (anonymous scraping, DataForSEO, Unwrangle) were evaluated.
